一：配置服务器监听器：

Listener.ora文件的例子
　　#listener.ora Network Configuration file: #c:\oracle\product\10.1.0\Db_2\NETWORK\ADMIN\listener.ora
　　# Generated by Oracle configuration tools.
　　#下面定义LISTENER进程为哪个实例提供服务
　　#这里是ORCL，并且它对应的ORACLE_HOME和GLOBAL_DBNAME
　　#其中GLOBAL_DBNAME不是必需的除非使用HOSTNAME做数据库连接

　　SID_LIST_LISTENER =
　　(SID_LIST =
　　(SID_DESC =
　　(SID_NAME = ORCL)
　　(ORACLE_HOME = c:\oracle\product\10.1.0\Db_1)
　
　　)
　　)
　　#监听器的名字，一台数据库可以有不止一个监听器
　　#再向下面是监听器监听的协议，ip，端口等，这里使用的tcp1521端口，并且使#用的是主机名
　　LISTENER =
　　(DESCRIPTION =
　　(ADDRESS = (PROTOCOL = TCP)(HOST = 主机名 )(PORT = 1521))
　　)



1：添加监听器：
打开：oracle net configuration assistant,选择监听器配置，添加，输入监听器名（不要重复），协议：tcp，端口：默认为1521，但不能重复使用，配置完成。

2：检查和修改监听器配置：使用oracle net manager
打开oracle  net manager ，选择刚才配置的监听器，address 有：protocol 协议，host主机，Port端口，可以修改，选择listening locations ，选数据库服务，输入：数据库全局名，地址，sid等，最后保存文件，保存网络配置。
3：启动和停止监听器
创建完后要先关闭监听器，重新以oracle用户启动
开始：运行：lsnrctl
可以用help查看lsnrctl的相关命令：
输入：stop listener
输入：Start listener
最终：监听程序不支持服务
      执行命令成功
监听器只能在服务端配置；

二：配置客户端连接


用本地命名配置单台客户机：3中方法
oracle net manager;
oracle net configuration asssistant;
手工配置文件：tnsname.ora和sql.net文件，
目录：
D:\oracle\product\10.2.0\client_1\NETWORK\ADMIN
在一台客户机上配置好后只要把相关的tnsname.ora和sql.net文件复制到其他客户机的oracle\product\10.2.0\client_1\NETWORK\ADMIN文件夹中即可远程连接服务机。
一般tnsname.ora在客户机上，如要多台服务器共享，则每台服务器都要定义tnsname.ora。
在用SQL*PLUS连接时，主机字符串中输入客户机中设置好的服务器名/网络服务名即可：
如：conn scott/orcl @zyr
